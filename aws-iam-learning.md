# IAM User and Console Learning

## Step-by-Step Process

1. Logged into AWS Console.
2. Opened IAM (Identity and Access Management) service.
3. Explored the IAM dashboard and interface.
4. Opened IAM Users section.
5. Created a new IAM user.
6. Enabled AWS Management Console access for the IAM user.
7. Set password for IAM user login.
8. Attached AdministratorAccess policy to the user.
9. Created the IAM user successfully.
10. Opened IAM user login link in Incognito mode.
11. Logged in using IAM user credentials.
12. Reset IAM user password during first login.
13. Successfully accessed AWS Console using IAM user.

---

# IAM Concepts Learned

1. Difference between Root Account and IAM User.
2. Difference between Policies and Roles.
3. Purpose of IAM User Groups.
4. Purpose of Access Analyzer.
5. Purpose of Identity Providers.
6. Purpose of Root Access Management.
7. Explored and understood the left-side IAM interface sections and their basic functions.

---

# Learnings

1. IAM users are safer than using the root account daily.
2. Policies define permissions.
3. Roles provide temporary access using permissions.
4. IAM is an important security service in AWS.
5. AWS security and access management are based heavily on IAM.
